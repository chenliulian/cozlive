## 核心服务

### 1. AI Agent Engine (Python/FastAPI)

**职责**: 提供 AI Agent 核心能力

**核心模块**:

| 模块 | 功能 | 技术 |
|------|------|------|
| 人格一致性引擎 | 确保 Agent 人设一致性 | 大五人格模型 |
| 情感计算引擎 | 拟人化情感互动 | 情感分析模型 |
| 长效记忆引擎 | 分层存储与检索 | 向量数据库 |
| 自主行为引擎 | Agent 自主社交 | 定时任务调度 |
| 进化学习引擎 | 持续学习与进化 | 强化学习 |

**API 端点**:
- `/api/v1/agents` - Agent 管理
- `/api/v1/chat` - 对话服务
- `/api/v1/memory` - 记忆服务
- `/api/v1/evolution` - 进化服务

### 2. User Service (NestJS)

**职责**: 用户认证、用户信息、个人资料

**功能**:
- 用户注册/登录/认证
- 用户资料管理
- 社交关系管理
- 会员等级管理

### 3. Social Service (NestJS)

**职责**: 社交关系、连接管理

**功能**:
- 轻触即连机制
- 关注/好友关系
- 社交图谱
- 亲密度计算

### 4. Content Service (NestJS)

**职责**: 内容发布、互动

**功能**:
- 动态发布（文字/图片/视频）
- 点赞、评论、转发
- 内容推荐算法
- AI 辅助创作

### 5. Community Service (NestJS)

**职责**: 社群管理

**功能**:
- 一键生成社群
- Agent 自主运营
- 场景化社交
- 社群活动

### 6. Message Service (NestJS)

**职责**: 实时消息、通知

**功能**:
- WebSocket 实时通信
- 消息持久化
- 离线消息推送
- 消息已读回执

## 数据存储

### 关系型数据库 (PostgreSQL)

**存储内容**:
- 用户信息
- 社交关系
- 社群数据
- 会员信息

**分库分表策略**:
- 按用户 ID 分片
- 读写分离

### 文档数据库 (MongoDB)

**存储内容**:
- 聊天记录
- 动态内容
- 操作日志
- Agent 配置

### 缓存 (Redis)

**缓存策略**:
- 用户会话
- 热点数据
- 限流计数
- 分布式锁

**数据结构**:
- String: Token、配置
- Hash: 用户信息
- List: 消息队列
- Set: 在线用户
- Sorted Set: 排行榜

### 向量数据库 (Milvus/Pinecone)

**存储内容**:
- Agent 记忆向量
- 用户偏好向量
- 内容嵌入向量

**索引策略**:
- IVF_FLAT: 精确搜索
- HNSW: 近似搜索

### 对象存储 (MinIO)

**存储内容**:
- 用户头像
- 动态图片/视频
- Agent 形象

## AI Agent 技术架构

### 人格一致性引擎

```python
class PersonalityEngine:
    """
    基于大五人格模型的人设一致性引擎
    """
    
    def __init__(self):
        self.dimensions = {
            'openness': 0-100,           # 开放性
            'conscientiousness': 0-100,  # 责任心
            'extraversion': 0-100,       # 外向性
            'agreeableness': 0-100,      # 宜人性
            'neuroticism': 0-100,        # 神经质
        }
    
    def generate_response(self, profile, message):
        # 1. 创建系统提示词
        system_prompt = self._create_system_prompt(profile)
        
        # 2. 调用 LLM 生成响应
        response = llm.chat(system_prompt, message)
        
        # 3. 验证人设一致性
        is_valid, score = self.validate_consistency(
            profile, response
        )
        
        # 4. 如不一致，重新生成
        if not is_valid:
            response = self._regenerate(profile, message)
        
        return response
```

### 长效记忆引擎

```
记忆分层架构:

┌─────────────────────────────────────┐
│         工作记忆 (Working)           │
│    当前对话上下文 (最近 10 轮)        │
├─────────────────────────────────────┤

│    人设基础、价值观 (不可变)          │
└─────────────────────────────────────┘
```

## 安全架构

### 认证授权

- **JWT Token**: 访问令牌 + 刷新令牌
- **OAuth 2.0**: 第三方登录
- **RBAC**: 基于角色的权限控制

### 数据安全

- **传输加密**: TLS 1.3
- **存储加密**: AES-256
- **敏感字段**: 单独加密存储

### 内容安全

- **全链路风控**: 实时内容审核
- **敏感词过滤**: Trie 树算法
- **图像审核**: AI 模型识别

## 监控与日志

### 监控指标

- **应用指标**: QPS、延迟、错误率
- **业务指标**: 日活、留存、转化率
- **资源指标**: CPU、内存、磁盘、网络

### 日志收集

- **结构化日志**: JSON 格式
- **日志级别**: DEBUG、INFO、WARN、ERROR
- **链路追踪**: OpenTelemetry

### 告警策略

- **P0**: 服务不可用，立即告警
- **P1**: 性能下降，5分钟内告警
- **P2**: 异常增长，30分钟内告警

## 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| FastAPI | 高性能异步框架，自动生成文档 |
| NestJS | 企业级 Node.js 框架，依赖注入 |

| Redis | 高性能缓存，支持多种数据结构 |
| Kafka | 高吞吐消息队列，持久化存储 |
| Milvus | 开源向量数据库，支持十亿级向量 |
| Kubernetes | 容器编排，自动扩缩容 |
